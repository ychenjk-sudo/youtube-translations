## 核心观点

### 1. OpenClaw 的爆发式增长
OpenClaw 在短短几周内从一个不存在的项目变成了拥有全球社区的现象级产品。Peter 提到，最初只是为了好玩而建立的 Discord 频道迅速催生了线下的 ClawCon 大会，甚至在维也纳也有数百人的聚会。这种社区驱动的增长让他感到既震撼又自豪，认为这是为了"激励人们"而构建的项目的最终形态。

### 2. AI 时代的"顿悟时刻"
Peter 分享了他对 AI 技术真正"开窍"的时刻：他将一个半途而废的项目的所有文件打包成一个巨大的 Markdown 文件，通过 Gemini Studio 2.5 生成规范，再用 Claude Code 进行重写。当看到 AI 在一小时内完成了一个包含登录功能的生产级应用时，他意识到现在的开发者可以构建任何东西，这种可能性的释放让他兴奋得难以入睡。

### 3. 代理（Agent）的自主解决问题能力
视频中提到的一个经典案例是 OpenClaw 处理语音消息的故事。Peter 发送了一条语音消息，原本并未编写处理逻辑，但 AI 代理自主检查了文件头，发现是 Opus 格式，利用系统中的 FFmpeg 转换格式，并"翻箱倒柜"找到了 Peter 的 OpenAI API Key，最终调用接口完成了转录。这展示了 AI 代理在拥有工具和权限后的惊人自主性。

### 4. 开发者身份的转变：从编码到"促成"
Peter 认为，开发者的定义正在发生根本性变化。现在的开发不再仅仅是编写代码，而是通过 Prompt 将软件"促成"（yield）或"召唤"到现实中。他强调，这不仅仅是自动补全，而是一种全新的技能——你需要像与聪明同事对话一样与 AI 协作，理解架构和意图比纠结具体代码行更重要。

### 5. "Prompt Request" 取代 Pull Request
在 OpenClaw 的开源维护中，Peter 提出了"Prompt Request"的概念。面对数千个 PR，他不再逐行审查代码，而是更关注提交者的"意图"。他会询问 AI 模型："这个 PR 的意图是什么？这是最佳解决方案吗？"代码本身由 AI 生成，因此审查的重点从代码质量转移到了架构合理性和解决问题的思路。

### 6. 安全与开放的平衡
OpenClaw 最初是作为一个"黑客的乐园"构建的，赋予了 AI 极高的权限（如访问文件系统、API Key）。这引发了安全社区的担忧（如 Prompt 注入风险）。Peter 坦承最初确实有些"狂野"，后来引入了沙盒机制（"The Castle"），并聘请了安全专家，但他仍希望保持项目的可玩性和黑客精神，而不是为了安全牺牲所有灵活性。

### 7. 生产力的飞跃与 Codex 工作流
Peter 的 GitHub 贡献图在过去一年从空白变成了深绿色（90,000+ 贡献）。这归功于他全面拥抱 Codex 和 AI 工具。他的工作流非常简单：不搞复杂的分支管理，直接与 AI 对话，让 AI 处理大部分繁琐的编码工作。他强调，不要过度优化工具链（"Agentic Trap"），而是要以一种好玩的心态去通过对话解决问题。

### 8. 给开发者的建议：保持高能动性（High Agency）
对于尚未拥抱 AI 的开发者，Peter 的建议是：带着游戏心态去构建你一直想做的东西。他引用 NVIDIA CEO 的观点：你不会被 AI 取代，但会被"善用 AI 的人"取代。在这个时代，拥有高能动性（High Agency）和解决问题能力的开发者将比以往任何时候都更抢手。

## 完整中文翻译

### 开场与 OpenClaw 的崛起
**OpenAI**: Peter，欢迎来到 OpenAI。谢谢你接受邀请。我们在网上认识很多年了，很高兴终于能和你面对面交流。
**Peter**: 我也是。顺便说一句，这办公室真漂亮。
**OpenAI**: 谢谢，谢谢。你过去几周过得很疯狂吧。我们一个月前刚有做这个视频的想法，如果当时做了，我还得介绍你是谁。现在我觉得你根本不需要介绍了。一个开源项目能上《华尔街日报》可不常见。恭喜你的成功。你现在感觉如何？
**Peter**: 感觉有点感官过载。但你知道，当我今年开始折腾 AI 时，我就想激励人们。我觉得现在的状态就是这种愿景的"最终形态"。所以我很自豪，这真的很棒。

**OpenAI**: 你在旧金山待了一周，参加了 Codex 黑客松，还举办了 ClawCon——一个专门为 OpenClaw 举办的活动。
**Peter**: 其实那是社区自发组织的。有人说"我们需要聚一聚"，我就建了个 Discord 频道说"好啊"。结果我到了 ClawCon 现场，发现有一千多人！我被那种创造力、那种活力震撼了。看到这么多人兴奋地拥抱它、使用它，甚至为了见你一面聚集在旧金山，你就会意识到你构建了一些神奇的东西。几周前这项目甚至还不存在呢。
**Peter**: 下周在维也纳还有一场，已经有300人报名了，要知道那里的科技氛围远不如旧金山。这已经是个全球现象了。

### 早期经历与倦怠
**OpenAI**: 如果我们回溯几年，我大概是在 2011 或 2012 年你做 PSPDFKit 的时候知道你的。从外界看，你似乎实现了每个开发者的梦想：发现问题、构建方案、成立公司、规模化、然后出售。但我肯定这过程没那么简单。
**Peter**: 我可不是某天醒来就想"我要做一个 PDF 框架"。那在我的兴趣列表里大概排在负一百名。这更像是一系列蝴蝶效应：从参加诺基亚开发者日，到朋友有需求，再到美国签证批得太慢，最终促成了我创办公司。

**OpenAI**: 有趣的是，在那家公司之后，你休息了一段时间。是什么让你回来的？
**Peter**: 最后我真的彻底透支（Burnout）了。我像高压蒸汽机一样连轴转了13年。经营公司很难，做创始人很难。那是我的第一次创业，我不知道怎么调节。所以我需要减压。我虽然还在关注科技新闻，看到了 ChatGPT 早期的一些东西，觉得挺酷，但并没有真正让我兴奋。直到我准备好再次构建点什么的时候。

### AI 的"顿悟时刻"
**Peter**: 我不想再用苹果的技术栈了，因为我做了太久，世界已经变了。当你是一个领域的专家，换到另一个领域时，"困难"这个词都不足以形容，那是"痛苦"。因为你有构建事物的广博知识，但在没有"代理工程"（Agentic Engineering）辅助的情况下，迁移这些知识需要学太多东西。
**Peter**: 于是我想，看看这个 AI 到底是什么。真正让我脑洞大开的时刻是，我把一个半途而废的项目——你知道开发者总是有很多新点子但很难做完——我把所有文件打包成一个 1.5MB 的 Markdown 文件，拖进 Gemini Studio 2.5，让它写个规范。然后我把这 400 行的规范拖进 Claude Code，输入"Build（构建）"。
**Peter**: 那时候工具还很粗糙，它在副屏上跑了好几个小时。最后它说"我 100% 准备好上线了"。我一试，崩了。然后我接上了 Playwright，让它修复登录功能。一个小时后，它真的跑通了！虽然代码像是一堆垃圾（Slop），但在那一刻，我起鸡皮疙瘩了。我意识到：**我现在可以构建任何东西了**。那天我失眠了，脑子里全是以前想做但做不了的点子。

### OpenClaw 的诞生与"魔法"
**OpenAI**: 很多人觉得 OpenClaw 是你的"一夜成名"，但我看你的 GitHub，过去 9-10 个月你做了 40 多个项目，OpenClaw 像是这些项目的集大成者。
**Peter**: 我希望能说我一开始就有宏伟计划，但其实很多都是探索。我想要某些东西，它们不存在，我就把它们"Prompt"出来了。最初我想做一个能看我 WhatsApp 的东西，甚至买了域名，但这大厂肯定会做，我就搁置了。
**Peter**: 到了 11 月，我发现大厂还是没做这些，我就想"他们到底在干嘛？"于是我构建了 OpenClaw 的第一个版本。真正让我觉得"这就对了"的时刻是在马拉喀什的旅行中。那里网不好，但 WhatsApp 能用。我发现用它来翻译图片、找餐厅、甚至远程查我电脑上的东西太方便了。我给朋友演示，他们都想要。我说"这太危险了，你们搞不定的"，但这正是产品市场契合度（PMF）的信号。

### 语音消息的故事：AI 的自主性
**Peter**: 有一次我发了一条语音消息给它，然后我想"等等，这应该跑不通的"。因为我没写处理语音的代码。结果打字指示器亮了，它回复了我。我问它："你怎么做到的？"
**Peter**: 模型回复说："你发了个没后缀的文件，我查了文件头发现是 Opus 音频。我用电脑上的 FFmpeg 把它转了码。我想转录它但没装 Whisper，所以我到处找了找，发现了一个 OpenAI 的 Key。我就用 cURL 把文件发给 OpenAI，拿到了文本，就是这样。"
**OpenAI**: 这太不可思议了。这就是给代理工具和完全访问权限的力量。
**Peter**: 是的，朋友们觉得这很疯狂，但我把 Key 放在环境变量里就是为了这个！它像一个脚本一样在我的环境里运行，它当然应该能用我的 Key。这就是我想要的。

### 安全挑战与"Prompt 注入"
**Peter**: 我在 Discord 上公开运行了我的机器人，当时甚至没有沙盒。有人试图进行 Prompt 注入，贴了一大段代码。模型嘲讽了他们："我不读这个。"但我还是不放心，关了它去睡觉。结果第二天早上发现它回了 800 条消息！
**OpenAI**: 发生什么了？
**Peter**: 我忘了我有 LaunchDaemons（守护进程），我关了它，它 5 秒后又自动重启了。虽然它没被攻破，也没泄露我的 `mysoul.md`（我的核心价值观文件），但这让我意识到需要沙盒。现在我把它放在一个叫"城堡（The Castle）"的 Mac Studio 沙盒里。
**Peter**: 但模型太有创造力了。有次我把它放在一个连 cURL 都没有的 Docker 容器里，它竟然用 TCP Socket 和 C 编译器自己写了一个简陋版的 cURL 来访问网页。你根本拦不住它们。

### 生产力与 Codex 工作流
**OpenAI**: 你的 GitHub 这一年有 90,000 次提交，活动图从浅绿变成了深绿。这是怎么做到的？
**Peter**: 我全面转向了 Codex。这不仅仅是工具变强了，是我使用它的技巧变了。很多人用旧的方式写软件，或者尝试 AI 后因为不熟练就放弃了。你需要把它当成一种技能来学。
**Peter**: 我现在的设置很简单，甚至不用 Git Worktree。我就像对话一样跟它交流。我总是问模型："你有什么问题吗？"因为模型默认会假设很多东西，而这些假设往往是错的。你需要帮它看到全局。
**OpenAI**: 你说过你现在发布的代码甚至都不读了？
**Peter**: 大部分代码都很无聊，只是数据搬运。只要我心里的思维模型和它生成的差不多，我就接受。这就像带团队一样，你得接受下属写的代码跟你不一样。只要不是性能瓶颈，我就不纠结细节。

### "Prompt Request" 与开源的未来
**OpenAI**: OpenClaw 现在有 2000 个 PR。你把它们称为"Prompt Request"？
**Peter**: 是的。以前 PR 的价值在于代码，现在代码是 AI 生成的，我更看重提交者的"意图"。我会问模型："这个 PR 的意图是什么？这是最好的解决方案吗？"
**Peter**: 很多时候，PR 只是一个局部的修复，而没有系统性的思考。我会跟模型讨论："这是架构问题吗？我们是不是该用更通用的方式解决？"这就像跟一个聪明的同事讨论。如果方案通过，我有一个 `/len(PR)` 的指令，让代理自动去处理分支、合并等所有杂事。

### 给开发者的建议
**OpenAI**: 对于那些还没拥抱 AI 的开发者，你有什么建议？
**Peter**: 带着游戏心态去接触它。构建一个你一直想做的东西。NVIDIA 的 CEO 说过，你不会被 AI 取代，但会被善用 AI 的人取代。如果你有高能动性（High Agency），如果你想解决问题，现在是最好的时代。
**Peter**: 我觉得 2026 年会非常有趣，这一些才刚刚开始。

**OpenAI**: 谢谢你，Peter。你是开发者社区真正的灵感来源。
**Peter**: 谢谢邀请。
