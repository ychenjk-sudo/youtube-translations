## 核心观点
### 1. AI 是住在电脑里的“怪才朋友”
AI 不仅仅是工具，更像是一个住在你电脑里、聪明且足智多谋的“怪朋友”。你可以通过 WhatsApp 或 Telegram 等聊天软件与它交流，让它帮你处理各种任务。它能访问你的文件、终端、浏览器甚至智能家居，就像一个全能的远程管家。

### 2. 80% 的 App 将会消失
未来，绝大多数功能单一的 App（如健身记录、航班值机、待办事项）都将被 AI 取代。因为 AI 拥有你的全局上下文（知道你吃了肯德基、知道你的航班信息），它能直接通过 API 或浏览器操作完成任务，而不需要你手动打开一个个独立的 App。

### 3. Clawd：连接聊天软件与电脑的桥梁
Peter 开发的 Clawd (CL A WD) 是一个基于 CLI 的 AI 助手，它将聊天应用（如 WhatsApp）与本地电脑连接。它不仅能聊天，还能调用 CLI 工具、控制浏览器、访问文件系统，甚至自我编程和修复 Bug。

### 4. AI 的惊人执行力与“自主性”
视频中展示了 AI 的强大能力：它能自动读取推特上的 Bug 报告，定位本地代码库，修复 Bug，提交代码，并回复推特用户。它甚至能处理未预设的语音消息格式，自动寻找工具（ffmpeg, whisper）进行转录，展现出极强的“资源整合能力”。

### 5. 编程范式的转变：从语法到思维
AI 让编程语言的语法变得不再重要。开发者可以轻松跨越技术栈（例如从 iOS 转到 Web 开发），因为核心的工程思维和系统设计能力才是关键。AI 能够处理具体的代码实现，让开发者专注于构建和创造。

### 6. 警惕“代理陷阱” (The Agentic Trap)
Peter 批评了当前过度复杂的 AI 编排系统（如 Gaston 等多智能体框架），认为它们往往是“垃圾生成器” (Slop Town)。他主张保持“人在回路” (Human-in-the-loop)，因为 AI 目前还缺乏“品味” (Taste) 和愿景，需要人类的直觉和审美来引导。

### 7. 通过“玩”来掌握 AI
不要仅仅通过一次测试就下定论。要真正掌握 AI，必须通过大量的“玩耍”和互动，去理解模型的思维方式、怪癖和语言。只有通过不断的试错和磨合，才能写出好的 Prompt，发挥 AI 的最大潜力。

### 8. Pull Request 即 Prompt
AI 让非技术人员也能参与软件开发。Peter 的朋友（非程序员）通过 AI 生成代码并提交 Pull Request。虽然代码可能不完美，但这些 PR 传达了清晰的“意图”，资深开发者可以将其视为一种高级的 Prompt，在此基础上完善功能。

### 9. 上下文管理与“长记忆”
随着模型（如 Codex, GPT-4）上下文窗口的扩大，复杂的上下文管理技术变得不再必要。现在的模型可以容纳整个功能的开发讨论。同时，Clawd 具有持久记忆，能记住用户的偏好（如护照信息、生活习惯），用得越久越顺手。

### 10. 拒绝过度工程，回归简单对话
在开发 Clawd 新功能时，Peter 往往直接将 Discord 上的用户反馈截图或复制给 AI，通过对话来探讨解决方案，甚至让 AI 直接读取代码库并更新 FAQ。这种“直接对话”的方式比复杂的自动化流程更高效。

## 完整中文翻译

### 开场：住在电脑里的新朋友
**Peter**: 这就像是你有一个新的、有点怪但非常聪明且足智多谋的朋友，他就住在你的电脑里。当时我还在摩洛哥，有人发推特给我报了一个 Bug。我直接把那条推特的截图发到了 WhatsApp 上。它（AI）读取了推特，理解了那里有个 Bug。它检查了 Git 仓库，修复了它，提交了代码，然后回复了推特上那个人说“现在修好了”。

**Peter**: 这大概会让手机上 80% 的 App 消失。既然我有一个无限足智多谋的助手，它已经知道我在做错误决定、知道我在吃肯德基，我为什么还要用 MyFitnessPal 来记录食物呢？这些东西太全能了，甚至有点可怕。就像是解开了束缚的 ChatGPT。很多人没意识到，如果你给 AI 访问你电脑的权限，它们基本上能做你能做的任何事。

**主持人**: 好的，欢迎大家。今天的嘉宾是 Peter，他是 Clawd (Claude) 的创造者，这是一个你可以直接在聊天软件里对话来处理事务的 AI 助手。今天 Peter 会向我们展示如何使用 Clawd。而且 Peter 对 AI 编程有很多非常棒的“暴论” (Hot Takes)，我很期待深入探讨。欢迎另一位 Peter。

**Peter**: 谢谢邀请。

### 什么是 Clawd？为什么是龙虾？
**主持人**: 很高兴见到你。我们先聊聊 Clawd 吧。它到底是从高层面上是做什么的？还有，为什么它是一只龙虾？（笑）

**Peter**: 是的。也许我得讲点背景故事。当我“退休”归来时，我想要一种能从手机上检查我电脑状况的方法。因为我完全投入到了这种“Vibe Coding”（氛围编程）的趋势中。你知道，你的 Agent 可能跑半小时，你去吃饭，或者它们跑了两分钟就停了，因为有个问题，然后你回去时会很烦。

**Peter**: 我本来没想自己做，因为我以为大厂都会做这个。这感觉太显而易见了吧？就像一种新的操作系统。但到了 11 月还没人做。我就想，好吧，我试个小的。这个“小东西”基本上就是把 WhatsApp 连接到 Claude 的代码接口。你发一条 WhatsApp 消息，它就带着 Prompt 打开二进制文件，然后返回结果。非常简单，一小时就写好了。

**Peter**: 结果它有了自己的生命。现在大概有 30 万行代码了。它支持地球上几乎所有的消息平台。虽然不是每一个，但我们在努力。我觉得这就是未来的方向：每个人都会有一个超级强大、伴随一生的 AI。事实证明，如果你给 AI 访问你电脑的权限，它们基本上能做你在电脑上能做的任何事。

**主持人**: 是的，而且到了这种程度，你不需要坐在那里像保姆一样盯着它，对吧？你给它一些指令，它就去干活，你回头检查一下就行了。

### 实战故事：自动修 Bug 与“幽灵”操作
**Peter**: 是的。我觉得这个项目既是技术也是探索，因为它是一个全新的品类。有一次我在摩洛哥参加朋友的生日旅行，我发现自己一直在用这东西。比如问路、找餐厅建议。

**Peter**: 有一天早上，有人发推特给我报了个 Bug。我直接截了个图发到 WhatsApp。它读了推特，理解了我的某个仓库里有 Bug。它检出了 Git 仓库，修复了它，提交了代码，然后回复了那个人说修好了。我就想，哇，这太棒了。

**Peter**: 还有一次，我在外面走，没法打字，就发了一条语音消息。我其实还没给它做语音消息的支持。结果它显示了“正在输入”。我想，咦，发生什么了？然后它就像什么都没发生一样回复了我。我就问：“你到底是怎么做到的？”

**Peter**: 它说：“嗯，我看到了一个文件，但没有后缀名（因为我没写这功能）。所以我看了文件头，发现是某种音频格式。我在你电脑上找到了 ffmpeg，把它转成了 wav 格式。然后我找 whisper.cpp，但你没装。不过我发现了这个 OpenAI 的 Key，所以我用 curl 把它发到了 OpenAI 的 API，拿到了转录文本，然后回复了你。”

**主持人**: 哇，太神了。

**Peter**: 是啊，这些东西太足智多谋了，虽然有点可怕。那一刻我顿悟了：这比在网页上用 ChatGPT 有趣多了。这是“解开束缚”的 ChatGPT。只要你给它工具，给它访问权限，它们就能解决任何问题。

### CLI 军团与生活黑客
**Peter**: 过去几个月，我建立了一支 CLI 军队。因为 Agent 擅长什么？调用 CLI。所以我写了访问 Google 的 CLI（包括 Google Places API），写了一个查表情包和 GIF 的，这样它能发表情包斗图。（笑）

**Peter**: 我甚至写了一个可视化声音的，想让它“体验”音乐。这有点艺术向了。我还写了一个黑进了本地外卖平台的，它能告诉我外卖还要多久。还有一个逆向工程了 Eight Sleep（智能床垫）的 API，所以它能控制我床的温度。

**主持人**: 真的吗？那你做这些的时候，是让 AI 帮你写代码吗？

**Peter**: 最有趣的是，在我的上一家公司，我是 iOS 和 macOS 专家，搞了 20 年 Apple 生态。但我回来后想，我受够了 Apple 的封闭，我想做 Web 应用。但我发现，当你擅长某样东西时，切换技术栈非常痛苦，你会觉得自己像个白痴。你要查每一个小细节，比如“什么是 prop”或者“怎么分割数组”。

**Peter**: 但有了 AI，这些痛苦都消失了。你可以应用你的系统级思维、架构能力，甚至你的“品味”，而不用纠结语法。这感觉像超能力。突然间我觉得我可以构建任何东西。语言不再重要，重要的是我的工程思维。

### 安装与配置 Clawd
**主持人**: 我们回到 Clawd。你能演示一下怎么安装吗？需要很懂技术吗？

**Peter**: 既需要也不需要。这个项目吸引了很多不太懂技术的人，因为它通过聊天界面掩盖了复杂性。就像和一个住在你电脑里的怪才朋友聊天。

**Peter**: 它是用 TypeScript 写的，可以在 Mac、Linux 甚至 Windows 上运行。官网 claw.bot 上有一行安装命令。虽然看起来很极客，但它是开源的。安装后，它会引导你配置，连接 Telegram、Discord 等。

**Peter**: 你可以选择模型。虽然它支持所有模型，但我推荐 Anthropic 的 Opus，因为它有一种特殊的“幽默感”和灵魂。它是第一个让我觉得用起来很“好玩”的模型。我把它设置成会“吐槽” (Roast) 我。

**主持人**: 哈哈，它会根据你电脑里的东西吐槽你吗？

**Peter**: 对。（演示）看，它说：“你为了逃避现实卖掉了公司，结果你的顿悟是‘我该写更多软件’。你太沉迷于替代品，以至于给自己造了个朋友，因为调试代码比约会更有趣。”

**主持人**: 哇，太狠了。

**Peter**: 是的。我把它连接到了我的一切：邮件、日历、文件、灯光（Philips Hue）、Sonos 音响。它甚至能叫我起床。

### 取代 App 的未来
**主持人**: 普通人能用它干什么？除了删光电脑文件之外？（笑）

**Peter**: 每个人用法都不同。有人用它写 iOS App，有人用它管理 Cloudflare。但我认为，这东西会取代你手机上 80% 的 App。

**Peter**: 为什么我要用 MyFitnessPal 记账？我有这个全能助手，它知道我在吃垃圾食品。我只要发张照片，它就存入数据库，计算卡路里，然后吐槽我该去健身房了。为什么我要用待办事项 App？它能直接帮我追踪。为什么我要用航旅 App？它能帮我值机。

**Peter**: 我让它帮我在英国航空值机。这简直是图灵测试的高级版——操控浏览器在航空公司网站上值机。第一次它花了好久，在我的 Dropbox 里找到了护照，提取了信息，填进去，最后成功了。现在它几分钟就能搞定。

**主持人**: 没错。既然它有 API 权限，能直接做事，为什么还要点那些孤立的小 App 呢？

### 关于 AI 编程的“暴论”：警惕 Agentic Trap
**主持人**: 你写过一篇文章叫《Just Talk To It》（就跟它说话）。现在推特上大家都在搞各种复杂的 Agent 框架。你的观点是什么？

**Peter**: 我称之为“代理陷阱” (The Agentic Trap)。人们发现 Agent 很棒，就想让它们做更多，于是掉进了兔子洞，构建了极其复杂的工具，但实际上并没有提高生产力。

**Peter**: 比如 Gaston 这种复杂的编排器，里面有“市长”、“监督者”等各种角色，我称之为“垃圾生成器” (Slop Town)。或者那种死循环运行的 Agent，如果一步错了就丢弃所有上下文重来，简直是 Token 燃烧机。

**Peter**: 这些 Agent 缺乏“品味”。如果你没有愿景，不问对的问题，它们生成的代码就是垃圾。我开发时，是有一个模糊的想法，然后在与 AI 的互动中逐渐清晰。如果你试图一开始就写好所有 Spec，你就失去了这种“人机循环”的魔力。

### 如何学习 AI：去“玩”
**Peter**: 很多人因为怀疑 AI，就忽略它一年，然后花一天时间测试，写个简单的 Prompt 让它做一个 iPhone App，结果失败了，就说“AI 不行”。

**Peter**: 这不对。你需要去“玩”，去理解这些“小怪兽”是如何工作的，理解它们的语言和推理方式。你会逐渐变得更好。这是一种新的技能。

**Peter**: 有时候它不工作，你可以问它“为什么你没做这个？”它会解释它的假设。这就像一种新的沟通艺术。

### 结语
**主持人**: 总结一下，不要迷信那些自动化的“垃圾生成器”，保持人类在循环中，因为人类提供品味和愿景。要去探索，去犯错，去建立自己的路径。

**Peter**: 没错。

**Peter**: claw.bot，或者 GitHub 上。拼写是 C-L-A-W-D，就像龙虾的钳子 (Claw)。
